# 二 编码(Base64): 
## (1)功能:
- Base-64编码将二进制字符串、文本或国际字符表示的数据转换为可移植的字母方便传输，然后再在远端解析出原始字符串.
- Http的基本验证：用户名和密码通过base-64编码后传输.
- http首部中的一些非法字符可以使用base64进行处理.

# 三 摘要： 
## (1)功能: 
- 对所有数据提取指纹信息，从而实现数字签名、数据完整性校验等功能.

## (2)摘要算法：
- SHA(Secure Hash Algorithm)：安全哈希算法，产生一个160位的信息摘要.
- MD5(Message Digest Algorithm)：消息摘要算法，产生一个128位的信息摘要.

## (3)相关类:
- java.security.MessageDigest.

# 四 密钥公钥 
## (1)两类密码体制： 
- 对称密钥密码体制：加密密钥和解密密钥相同，例如：DES.
- 公钥密码体制：使用不同加密密钥和解密密钥，例如：RSA.

# 五 数字签名: 
## (1)功能： 
- 保证消息传输的完整性、对发送者身份的验证.

## (2)原理：
- 发送者：使用摘要算法对发送报文进行消息摘要，然后用自己的私钥对摘要进行加密，然后加密后的摘要作为报文的数字签名和原始报文一起发送给接收方.
- 接收方：收到消息后，首先使用同样的摘要算法对原始报文进行信息摘要，然后使用发送方的公钥对报文的数字签名进行解密，若两个消息摘要相同，那么接收方就确认该报文是发送方发的.
